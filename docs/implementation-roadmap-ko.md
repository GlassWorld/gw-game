# RBB 구현 계획 문서

## 1. 문서 목적

이 문서는 `RBB (Random Boss Break)`를 실제로 확장해 나갈 때 필요한 작업 항목을 우선순위 기준으로 정리한 실행 계획 문서다.

기획 문서인 [prototype-plan-ko.md](/home/glassworld/workspace/gw-game/docs/prototype-plan-ko.md) 가 `무엇을 만들지`를 설명한다면, 이 문서는 `어떤 순서로 작업할지`를 정리한다.

## 2. 현재 상태 요약

현재 저장소에는 아래 범위가 이미 구현되어 있다.

- 메인 화면
- 캐릭터 선택
- 주사위 기반 보스 선택
- 대기소 화면
- 캐릭터별 스킬 3개 선택
- 스탯 / 포션 선택과 자원 반영
- 연습 공간
- 보스 진입 / 사망 컷씬
- 다중 보스 런 진행도
- 4종 보스 placeholder
- 보상 표시
- 보스별 전장 톤 / 텔레그래프 색 / 템포 차이
- 보스 전투
- 로딩 / 카운트다운 / 결과 화면
- 기본 HUD
- 우클릭 이동 / 자동 평타 / `Space` 대시 / `Q W E` 스킬

즉, 현재는 `런 구조를 갖춘 전투 코어 프로토타입` 단계다.

아직 없는 핵심 시스템:

- 유물 / 장비 / 강화 실제 시스템
- 보스별 고유 지형 / 기믹 / 패턴 차별화
- 보스 문서 추가 확장
- 기록 / 랭킹 시스템
- 백엔드 저장 구조

## 3. 우선순위 원칙

작업 순서는 아래 원칙을 따른다.

1. 전투 코어와 화면 흐름을 먼저 안정화
2. 그 다음 메타 루프를 붙인다
3. 데이터 구조를 먼저 만들고, 연출은 나중에 강화한다
4. 백엔드가 필요한 기능은 마지막에 검토한다

즉, `컷씬 퀄리티`보다 `선택 → 준비 → 연습 → 전투 → 복귀` 루프를 먼저 완성하는 쪽이 맞다.

## 4. 작업 단계

## Phase 1. 현재 프로토타입 안정화

목표:

- 이미 있는 전투 코어를 흔들리지 않게 만든다
- 선택한 캐릭터 / 스킬 / HUD / 실제 전투 결과가 항상 일치하도록 고정한다

작업 항목:

- [x] 전투 시작 흐름 점검
- [x] 연습 공간에서 선택값 반영 검증
- [x] 전투 HUD 최종 정리
- [x] 스킬 범위/시전/쿨타임 표시 일관화
- [x] 보스 패턴 속도와 텔레그래프 1차 밸런싱
- [ ] 캐릭터별 스킬 설명과 실제 동작 추가 검증
- [ ] 현재 버전 플레이 테스트 체크리스트 문서화

완료 기준:

- 지금 있는 단일 보스 프로토타입을 반복 플레이해도 흐름이 깨지지 않아야 한다

## Phase 2. 시작 화면과 기본 런 구조 정리

목표:

- 현재 메인 화면을 `RBB` 기준 구조로 정리한다
- 게임이 시작되면 자연스럽게 `캐릭터 선택 → 보스 선택`으로 이어지게 만든다

작업 항목:

- [x] 시작 화면을 `RBB + 시작하기` 중심으로 단순화
- [x] 캐릭터 선택 화면을 현재 구조 기준으로 정리
- [x] 보스 선택 화면 전용 컴포넌트 추가
- [x] 화면 전환 상태를 `home / character / boss / hub / practice / battle / result` 중심으로 재정리

완료 기준:

- 시작 버튼을 누른 뒤 전체 준비 흐름이 자연스럽게 이어져야 한다

## Phase 3. 주사위 기반 보스 선택 구현

목표:

- 리스트 선택이 아니라 `주사위가 굴러가며 보스가 정해지는 구조`를 만든다

작업 항목:

- [x] 보스 선택용 데이터 구조 정의
- [x] 주사위 결과 면과 보스 매핑 구조 정의
- [x] 주사위 애니메이션 UI 구현
- [x] 선택된 보스 확대 연출 추가
- [x] 보스명 / 지역명 표시 UI 추가
- [x] 이미 나온 보스를 제외하는 중복 방지 규칙 구현
- [x] 남은 보스 수 표시

완료 기준:

- 한 런에서 같은 보스가 다시 나오지 않아야 한다

## Phase 4. 대기소 시스템 구현

- [x] 세팅 완료 후 연습 공간으로 이동

## 5. 추천 바로 다음 작업

현재 기준에서 바로 다음으로 진행하는 것을 추천하는 항목은 아래 5개다.

1. 보스 4종 placeholder 문서를 `docs/bosses/`에 추가
2. 보스별 고유 패턴 또는 지형 차별화 시작
3. 연습 표적 다양화
4. 스킬 강화 규칙을 실제 시스템으로 연결
5. 캐릭터별 스킬 설명과 실제 동작 최종 검수

이 5개를 먼저 하면, 이후 대기소와 다수 보스 구조를 붙이기 쉬워진다.

## 6. 권장 작업 단위

한 번에 너무 크게 붙이지 말고 아래 단위로 자르는 것이 좋다.

- 작업 1: 시작 화면 정리
- 작업 2: 보스 선택 화면 뼈대
- 작업 3: 런 상태 데이터 구조
- 작업 4: 대기소 기본 레이아웃
- 작업 5: 보스 문서 템플릿

즉, `주사위 + 대기소 + 자원 + 컷씬`을 한 번에 하지 말고, 화면 흐름부터 단계적으로 붙인다.

## 7. 주의사항

- 현재 전투 코어를 불안정하게 만들면서 상위 구조를 붙이면 안 된다
- 아직 백엔드가 없으므로 기록/랭킹은 문서와 인터페이스 수준으로 먼저 정리한다
- 보스를 여러 종 추가하기 전에 보스 데이터 포맷을 먼저 정해야 한다
- 최종 이펙트나 고퀄 연출보다, 먼저 구조와 흐름을 맞춘다
