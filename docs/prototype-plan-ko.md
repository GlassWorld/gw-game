## 2. 게임 한 줄 설명

캐릭터와 스킬을 고르고, 매번 다른 보스를 주사위로 만나며, 기믹과 지형이 다른 보스들을 연속으로 돌파하는 보스 러시형 액션 게임이다.

## 3. 상위 게임 목표

이 프로젝트의 장기 목표는 단일 보스전 프로토타입을 넘어, 아래 구조를 가지는 반복형 보스 공략 게임으로 확장하는 것이다.

- 여러 캐릭터 중 하나를 선택
- 보스 주사위를 굴려 새로운 보스를 만남
- 대기소에서 스킬, 유물, 스탯, 포션, 장비를 한정 자원 안에서 세팅
- 연습 공간에서 현재 세팅을 시험
- 보스 컷씬 이후 실제 전투 진입
- 보스 처치 후 보상 획득
- 다시 대기소로 돌아가 다음 보스를 선택
- 모든 보스를 클리어하면 게임 클리어

즉, `전투 1회`가 목적이 아니라 `랜덤하게 이어지는 여러 보스 공략 루프`가 최종 목표다.

## 4. 핵심 재미

- 보스 패턴을 읽고 대응하는 재미
- 지형과 기믹이 다른 보스를 만날 때의 긴장감
- 한정된 자원 안에서 세팅을 짜는 선택의 재미
- 연습 공간에서 조합을 시험하고 실전에 적용하는 재미
- 연속 클리어를 통해 점점 강해지는 빌드의 재미

## 5. 기본 세계관/구조 방향

RBB는 보스를 하나씩 선택해서 깨는 메뉴형 구조보다, `주사위로 다음 보스가 결정되는 흐름`을 핵심 구조로 삼는다.

- 보스마다 지역이 다르다
- 보스마다 기믹, 지형, 패턴, 컷씬이 다르다
- 이미 나온 보스는 같은 런에서는 다시 나오지 않는다
- 주사위의 모든 보스를 클리어하면 해당 런은 게임 클리어가 된다

## 6. 기본 화면 흐름

### 시작 화면

게임 시작 화면은 아주 단순하게 유지한다.

- 타이틀: `RBB`
- 버튼: `시작하기`

첫 화면은 설명을 길게 늘어놓기보다, 시작 버튼으로 바로 진입하는 구조를 우선한다.

### 전체 진행 순서

1. 시작 화면에서 `시작하기`
2. 캐릭터 선택 화면 진입
3. 보스 선택 화면 진입
4. 주사위가 굴러가고 윗면의 보스가 결정됨
5. 선택된 보스가 확대 연출로 강조됨
6. 보스 명칭과 지역이 표시됨
7. 대기소 진입
8. 대기소에서 스킬, 유물, 스탯, 포션, 장비를 세팅
9. 연습 공간에서 현재 세팅 테스트
10. 보스 진입
11. 보스 컷씬
12. 3초 카운트다운
13. 전투 시작
14. 보스 처치 시 보스 사망 컷씬
15. 대기소로 복귀
16. 주사위를 다시 던져 다음 보스 선택
17. 모든 보스를 클리어하면 게임 클리어

## 7. 캐릭터 선택

현재 캐릭터 라인업은 아래 3종을 기준으로 잡는다.

- A 검사
- B 마법사
- C 성기사

캐릭터 선택 화면의 목적은 단순한 외형 선택이 아니라, 이후 스킬 풀과 운용 방식이 달라지는 첫 번째 전략 선택이다.

캐릭터마다 달라질 수 있는 요소:

- 기본 공격 방식
- 이동/생존 성향
- 스킬 5종 풀
- 추천 세팅 방향
- 특정 보스 상성

## 8. 보스 선택 구조

### 주사위 기반 선택

보스 선택은 일반적인 리스트 선택이 아니라 `주사위 연출` 기반으로 진행한다.

- 주사위가 굴러간다
- 최종적으로 나온 윗면이 이번 보스가 된다
- 결과 보스는 화면에서 크게 확대된다
- 보스 이름과 지역명이 함께 표시된다

### 중복 방지

한 런에서 이미 선택되었거나 클리어한 보스는 다시 나오지 않는다.

- 이미 나온 보스 면은 다음 굴림에서 제외
- 남은 면 중에서만 새 보스가 결정
- 모든 면을 클리어하면 런 종료

이 규칙은 `반복 회차의 진행감`을 만드는 핵심 장치다.

## 9. 대기소 설계 방향

대기소는 단순 메뉴가 아니라, 다음 보스를 준비하는 `전투 준비 허브` 역할을 한다.

대기소에서 할 수 있는 것:

- 캐릭터 스킬 선택
- 유물 선택
- 스탯 배분
- 포션 세팅
- 장비 세팅
- 현재 보유 자원 확인
- 다음 전투 준비 완료

### 자원 기반 선택

모든 것을 다 챙기는 구조가 아니라, 한정된 자원을 사용해 필요한 것을 골라야 한다.

예시:

- 더 강한 스킬을 살지
- 생존용 포션을 챙길지
- 유물을 우선할지
- 공격 스탯에 투자할지

즉, 대기소는 `전략 선택 공간`이어야 한다.

## 10. 연습 공간

연습 공간은 대기소 이후, 실제 보스 진입 전에 현재 세팅을 검증하는 공간이다.

연습 공간의 목적:

- 선택한 스킬 확인
- 스킬 범위/쿨타임 체감 확인
- 장비/유물 조합 테스트
- 조작 적응

연습 공간에서 충분히 시험한 뒤 실제 보스로 진입하는 구조를 기본으로 한다.

## 12. 보상과 성장 구조

보스를 클리어하면 아래 보상을 획득한다.

- 약간의 재화
- 스킬 강화

이 보상은 다음 보스를 준비하는 대기소 단계에서 다시 사용된다.

장기적으로 고려할 수 있는 성장 축:

- 스킬 레벨
- 장비 강화
- 포션 품질
- 유물 조합 확장
- 캐릭터별 특화 트리

## 13. 게임 클리어 조건

- 주사위에 배치된 모든 보스를 클리어하면 게임 클리어

즉, 단일 보스를 잡는 것이 아니라 `한 런의 모든 보스를 순회 클리어`하는 구조다.

## 14. 기록과 랭킹

장기 목표로 대시보드 시스템을 둔다.

대시보드에 포함할 항목:

- 플레이어 이름 기록
- 보스별 클리어 시간 기록
- 클리어 타임 기준 랭킹

랭킹은 최종 구현 시 백엔드/저장 구조가 필요하므로, 현재 프론트엔드 프로토타입 범위를 넘어서는 시스템이다. 다만 기획 상 중요 항목으로 먼저 정의해둔다.

## 15. 현재 프로토타입과의 관계

현재 저장소에 구현된 것은 `RBB의 최소 전투 프로토타입`이다.

현재 실제 구현 범위:

- 메인 화면
- 캐릭터 선택
- 주사위 기반 보스 선택
- 보스 4종 placeholder 등록
- 캐릭터별 5개 스킬 중 3개 선택
- 대기소 세팅 화면
- 스탯 / 포션 선택과 자원 표시
- 연습 공간
- 보스 진입 컷씬 / 사망 컷씬
- 보스 전투
- 로딩 / 카운트다운 / 결과 오버레이
- 보스 패턴 회피형 전투
- 클리어 보스 제외 규칙
- 보상 표시
- 모든 등록 보스 클리어 시 런 종료

즉, 지금 구현은 `RBB 전체 구조의 전투 코어 검증 버전`으로 이해해야 한다.

아직 미구현인 상위 시스템:

- 유물 / 장비 / 강화 실제 시스템
- 보스별 고유 지형 / 기믹 / 패턴 심화
- 보스별 지역 상호작용
- 랭킹 / 기록 시스템
- 백엔드 저장 구조

## 18. 현재 기준 메모

- 현재 프로젝트는 Nuxt3 + TypeScript + Phaser 기반이다
- 전투는 프론트엔드만으로 동작한다
- SSR 문제를 피하기 위해 Phaser는 전투 DOM 생성 후에만 마운트한다
- 지금 단계에서는 최종 이펙트보다 구조, 전투 루프, 선택 흐름, 데이터 분리가 더 중요하다
